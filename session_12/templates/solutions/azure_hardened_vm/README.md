# Azure Hardened VM — CCI Session 12, Lesson 9 (runbook)

The full lifecycle of a server in the cloud: **create a resource group → harden a
VM → lock the firewall → serve a page → verify → delete everything.** Your
instructor runs this live; this is the numbered runbook you keep and replay later
when you have your own Azure access.

Everything runs in **Azure Cloud Shell** — a terminal in your browser
(<https://shell.azure.com>) with the `az` tool already installed and logged in.
**Nothing is installed on your laptop.**

## The lifecycle

```
01  Resource group     ./01_resource_group.sh     a labelled box that holds everything
02  Hardened VM         ./02_create_vm.sh           SSH-key login, non-root user, patched image
03  Lock the firewall   ./03_lock_nsg.sh            open ONLY 443 (public) and 22 (you); shut the rest
04  Serve a page        ./04_serve_site.sh          nginx + site/index.html
05  Verify              ./05_verify.sh              page loads; audit the open doors
99  DELETE EVERYTHING   ./99_delete_everything.sh   erase the whole box — DO NOT SKIP
```

## How to run

1. Open <https://shell.azure.com> and choose **Bash**.
2. Upload this `azure_hardened_vm/` folder (Cloud Shell has an upload button) or
   recreate the scripts there.
3. Run the scripts **in order**:
   ```bash
   chmod +x *.sh
   ./01_resource_group.sh
   ./02_create_vm.sh
   ./03_lock_nsg.sh
   ./04_serve_site.sh
   ./05_verify.sh
   # ... when you're done looking at it:
   ./99_delete_everything.sh
   ```
4. Change settings (names, region, VM size) in **`config.sh`** — every script reads
   from there.

## The security ideas, one per step

- **Resource group** — one box per experiment, so teardown is a single command.
- **VM** — `--generate-ssh-keys` (log in with a key no one can phish, not a
  password), a **non-root** admin user (least privilege), a **patched** image.
- **NSG** — the reception desk: **open the minimum**. 443 to the public, 22 to your
  IP only, everything else shut. See `nsg_rules.md`.
- **Serve** — `sudo` lets a least-privilege user borrow admin rights for one command.
  (Real clinical traffic must be HTTPS/443 with a certificate — Lesson 2.)
- **Verify** — confirm the open door works *and* that the closed doors are closed.
  Then ask an AI: *"what's still exposed?"*
- **Delete** — a running VM is a bill that never sleeps and a 24/7 attack surface.
  The most secure server is one that no longer exists.

## Safety rules (non-negotiable)

- **No secrets in any file.** The SSH key is generated by Azure and stays in Cloud
  Shell. Nothing here contains a password or key.
- **Never open a login/database port to `0.0.0.0/0`** "just to make it work." That
  is the shortcut attackers count on.
- **Always end with `99_delete_everything.sh`.** The lab is not complete until
  `az group list` shows the box is gone.
