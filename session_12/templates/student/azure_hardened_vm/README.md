# Azure Hardened VM — CCI Session 12, Lesson 9 (STUDENT runbook)

This is the **student** copy. Your instructor runs the full lab live; you keep this
runbook to replay later with your own Azure access. It has one `TODO` in
`03_lock_nsg.sh` (the "open the minimum" firewall step) — see `STUDENT_TASKS.md`.
The complete scripts are in `../../solutions/azure_hardened_vm/`.

Everything runs in **Azure Cloud Shell** (<https://shell.azure.com>) — nothing
installed on your laptop.

## The lifecycle (run in order)

```
chmod +x *.sh
./01_resource_group.sh     # a labelled box that holds everything
./02_create_vm.sh          # SSH-key login, non-root user, patched image
./03_lock_nsg.sh           # open ONLY 443 (public) + 22 (you)  <-- has a TODO
./04_serve_site.sh         # nginx + site/index.html
./05_verify.sh             # page loads; audit the open doors
./99_delete_everything.sh  # erase the whole box — DO NOT SKIP
```

Change names/region/size in `config.sh`. Read `nsg_rules.md` for the firewall logic.

## Safety rules (non-negotiable)

- **No secrets in any file** — the SSH key is generated by Azure, stays in Cloud Shell.
- **Never open a login/database port to `0.0.0.0/0`.** Open the minimum.
- **Always end with `99_delete_everything.sh`** — a running VM is a bill and an
  attack surface. The lab isn't done until `az group list` shows the box is gone.
