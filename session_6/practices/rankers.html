RETRIEVE → RERANK → ANSWER

Retrieval is recall. Ranking is precision. Vector search and graph traversal grab fifty plausible candidates; the ranker decides which five actually get in front of the model. Pick the wrong ranker and a clinical RAG system confidently surfaces a 2008 protocol over today's standard of care.

N° 01 — THE WORKHORSE

Cross-Encoder
Rerankers

Encodes the query and each candidate together for direct relevance scoring. Roughly 100× slower than a bi-encoder, far more accurate. Run only on the top 50–100 retrieved items.
COHERE RERANK · JINA-V2 · BGE-RERANKER · MS-MARCO MINILM
PRODUCTION α
N° 02 — THE ORACLE

LLM-as-
Reranker

Hand the model 20 candidates, ask for a ranked list. Pointwise, pairwise, or listwise (RankGPT). Highest accuracy you can buy today; also the most expensive and the slowest.
RANKGPT · POINTWISE · PAIRWISE · LISTWISE
HIGH-STAKES β
N° 03 — THE FUSER

RRF

Reciprocal Rank Fusion merges results from multiple retrievers without needing comparable scores. The bedrock of every hybrid search system.
VECTOR + BM25 · FREE · DETERMINISTIC
FREE ⊕
N° 04 — THE EDITOR

MMR

Maximal Marginal Relevance balances relevance with novelty. Stops the LLM from reading the same fact rephrased five times. One line in LangChain. Underrated.
DIVERSITY · ANTI-REDUNDANCY · CHEAP
EASY WIN ∇
N° 05 — THE GATEKEEPER

Metadata
Filters

Often the simplest "ranker" is a hard filter: date, source authority, specialty, language. A 2008 Wilms regimen should not outrank the current SIOP protocol.
DATE · SOURCE · SPECIALTY · LANGUAGE
CRITICAL ◇
N° 06 — THE NETWORKER

Graph-Aware Reranking

Specific to GraphRAG: rerank candidate nodes by graph features — PageRank centrality, path-length to query-mentioned nodes, Louvain community match. This is where community detection finally pays its rent.
PAGERANK · CENTRALITY · COMMUNITY · PATH-LENGTH
GRAPHRAG ⌬
N° 07 — THE LEARNER

Learned-to-Rank

LightGBM or similar trained on user feedback — clicks, thumbs, helpfulness. Powerful long-term, useless on day one. Skip until you have six months of usage data.
LIGHTGBM · GBDT · FEEDBACK LOOP
